Website

Main Menu

The main menu shows all transit systems and some basic information about them. This corresponds to the / endpoint of the Data API.

Log In

The /login page (accessed with the "Log In" button) allows the user to log into a user account. This corresponds to the /login endpoint of the Auth API.

System Configuration

After clicking on one of the pen icons, the configuration for the given transit system is shown. If the user is logged in and authorized to edit this configuration, this page will allow that, otherwise the configuration can only be viewed. This corresponds to the /:system/config endpoint of the Config API.

When logged in, after clicking on the plus icon, the configuration editing page is shown, but here it allows adding a new transit system's configuration. This corresponds to the /new endpoint of the Config API.

The configuration page is a form, with a text input for the system's name, a map which functions as a position input for the system's initial map location, and inputs for GTFS Schedule sources and their associated GTFS Realtime sources.

System Overview

After clicking on one of the transit systems' names, a system overview page with a map is shown.

The map's sidebar contains all lines, stops, and current alerts for the given transit system.

The map itself shows all stops and all known vehicle positions.

This corresponds to the /:system/stops, /:system/lines, /:system/alerts, and /:system/vehicles endpoints of the Data API.

Line

After clicking on a transit line in the sidebar, an overview of its stops is shown, and its route (if known) is shown on the map.

This corresponds to the /:system/line/:id endpoint of the Data API.

Stop

After clicking on a transit stop on the map or the sidebar, the stop's icon on the map is highlighted and its arrivals and schedule (in separate tabs) are shown. These can be filtered by line name.

This corresponds to the /:system/stop/:id endpoint of the Data API.
